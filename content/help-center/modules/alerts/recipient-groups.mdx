---
title: Recipient groups & email setup
description: Manage who receives which alerts — create, edit, and delete recipient groups to route alert emails to the right teams.
---

Recipient groups decide who gets which alert email. Setting them up by team, priority, or client is the cleanest way to keep your <Term>Buy Box</Term> notifications out of the wrong inbox. This page covers when to create a group, how to set one up alongside an alert, and how to edit it later.

## When to use recipient groups

- **Different teams** — <Term>Buy Box</Term> alerts to your paid-media team, listing alerts to your content team
- **Different priorities** — top-seller alerts to leadership, full catalog alerts to operations
- **Per-client** — agencies can route alerts to each client's stakeholders

## Create a group

<Steps>

<Step>
### Open the Alerts module

Open **Apps → Alerts** and click **+ Alert**.
</Step>

<Step>
### Configure the alert basics

Select your triggers, click **Continue**, then pick the product(s) to be alerted on.
</Step>

<Step>
### Create a new alert group

Click **Create a new alert group**.

<img
  src="/images/help-center/modules/alerts/recipient-groups/recipient-groups-1.png"
  alt="Recipient Groups 1"
  width={1200}
  height={680}
  className="rounded-lg border"
/>
</Step>

<Step>
### Name the group and add members

Add the group name, then select the members of the <Term>workspace</Term> that should receive notifications.
</Step>

<Step>
### Name and save the alert

Name your alert, then click **Create Alert**.
</Step>

</Steps>

## Edit a group

</Step>

<Step>

## What the email looks like

Each daily email contains:

- A summary line per triggered event (max 20)
- If you have more than 20, you can go on "See more changes in app (XXX total changes)"
- The product or tag the event relates to
- A link back to the relevant DataHawk report

## When emails are sent

DataHawk runs the trigger checks once per day and bundles all events into a single email.

Every day at 12pm CET.
